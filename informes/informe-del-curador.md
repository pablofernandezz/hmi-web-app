## Objetivos de aprendizaje de la semana 1

---

-   **Concepto: Diseño Mobile First y Adaptativo**

    El objetivo principal fue utilizar la metodología Mobile First, que invierte el flujo de trabajo tradicional al diseñar primero pantallas más pequeñas y escalar progresivamente. Implementamos un diseño adaptativo utilizando CSS Media Queries, lo que permite que la interfaz cambie su estructura sin usar JavaScrip, sólo con un único documento HTML.

    **Recursos identificados para su estudio:**

    *   MDN Web Docs: Guí sobre Responsive Design y Media Queries

    *   W3C Standards: Especificaciones sobre CSS y Box Model.


-   **Concepto: HTML semántico y Accesibilidad Web**

    Uso estricto de HTML5 semántico. Se separa completamente el contenido de la presentación y asegurar que la interfaz cumpla con pautas de accesibilidad.

    **Recursos identificados para su estudio:**

    *   Guías de Accesibilidad del W3C: Contrastes de color y uso de atributos ARIA.

    *   Baseline: Para asegurar que características de CSS y HTML son compatibles con navegadores modernos.

## Recursos empleados en la semana 1


---


-   **Recurso: Balsamiq**


    *   **Descripción:** Herramienta de software para la creación de wireframes de baja fidelidad, que permite prototipar rápidamente la estructura y el flujo de una interfaz de usuario.

    *   **Utilidad y aplicación a la práctica:** Primer paso antes de la codificación. Utilizado para definir la transformación entre dispositivos.

        1.  **Móbil:** Vista vertical de columna única.

        2.  **Tablet:** Vista dividida (Maestro-Detalle)

        3.  **Escritorio:** Vista tipo Dashboard con mayor espaciado.

---


-   **Recurso: Extensiones de VScode**


    *   **Descripción:** Extensiones (Live Server y WebHint) de entorno de desarrollo Visual Studio Code para desarrollo web

    *   **Utilidad y aplicación a la práctica:** Live Server nos permitió ver cambios en tiempo real y probar el comportamiento adaptativo redimensionando la ventana del navegador. Por otra parte, WebHint nos advierte de problemas de accesibilidad y compatiblidad con navegadores, cumpliendo así con los estándares exigidos.
---

-   **Recurso: Gemini (IA Generativa)**

    *   **Descripción:** Modelo de lenguaje grande desarrollado por Google, utilizado como asistente de desarrollo y conceptualización.

    *   **Utilidad y aplicación a la práctica:** Apoyo para consultar dudas puntuales, algunas correcciones en el código,...
